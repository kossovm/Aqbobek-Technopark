Спецификация проекта: Aqbobek Technopark LMS
1. Описание продукта и стек
Цель: Платформа для жесткого цифрового контроля инвентаря, расходных материалов, проектов и расписания в школьном технопарке. Решает проблему воровства, потери вещей и бесконтрольного расхода материалов через персональную ответственность.
Стек технологий: Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui, Supabase (PostgreSQL, Auth, Storage), Vercel.
Роли пользователей: Admin (Администратор), Teacher (Учитель), Student (Ученик), Guest (Неавторизованный пользователь).

2. База данных (Supabase SQL Архитектура)
Система должна опираться на следующие основные сущности:

users: Ученики и учителя (id, роль, статус ознакомления с правилами).

locations: Физические места хранения (Шкаф 1, Полка А).

inventory: Оборудование (уникальный ID AQB-INV-XXX, название, статус: свободен/занят, qr-код, привязка к локации).

transactions: Логи (кто взял, что взял, когда взял, когда вернул).

consumables: Расходники (пластик, фанера) с учетом остатка по весу (граммы) или площади (см²).

consumable_logs: Логи списания расходников со ссылкой на скриншот-подтверждение.

projects: Учебные проекты (название, статус).

project_inventory: Связующая таблица (какой инвентарь выделен под конкретный проект).

3. Ключевой функционал (Модули)
Модуль 1: Выдача и Возврат (Ядро системы)
Сканер QR-кодов: Интерфейс с доступом к камере устройства (через html5-qrcode).

Логика Взятия: Пользователь сканирует свободную вещь -> система спрашивает подтверждение -> вещь записывается на пользователя (или привязывается к его проекту). Статус вещи меняется на "Занято".

Логика Возврата: Пользователь сканирует вещь, которая числится на нем -> нажимает "Вернуть" -> система показывает подсказку: "Положите на место: [Название локации из БД]". Статус меняется на "Свободно".

Модуль 2: Учет расходных материалов (Consumables)
Списание: Ученик выбирает материал (например, "PETG пластик черный") и вводит потраченное количество (вес для 3D-печати, площадь для резки).

Доказательство (Storage): Обязательное поле для загрузки скриншота из слайсера (Orca, Bambu Studio), подтверждающего расход. Файл улетает в Supabase Storage.

Привязка: Списание привязывается к пользователю и/или конкретному проекту.

Модуль 3: Дашборд пользователя и Проекты
Мой инвентарь: Список всего, что сейчас числится на пользователе, с возможностью быстрого возврата (открытие сканера).

Мои проекты: Список активных проектов ученика.

Проектные коробки: Внутри проекта видно, какой инвентарь к нему привязан и на какой физической полке лежит "коробка проекта".

Модуль 4: Онбординг и База Знаний (Knowledge Base)
Правила: Раздел с инструкциями (работа с 3D-принтерами, техника безопасности, система штрафов за порчу и потерю).

Hard-блок: При первой авторизации пользователь не может перейти к сканированию или проектам, пока не поставит галочку "Я ознакомился с правилами и несу материальную ответственность". Этот флаг сохраняется в БД профиля.

Модуль 5: Админ-панель (Control Room)
Управление пользователями: Ручное добавление, подтверждение регистрации, бан нарушителей, сброс паролей.

Управление инвентарем: Добавление новых позиций, распределение по локациям.

Генератор этикеток: Автоматическая генерация уникального ID (например, AQB-001) и QR-кода. Кнопка "Печать", которая открывает системный диалог window.print() со стилями, адаптированными под термопринтер HM-T260 (или аналог 30x20 мм).

Аналитика: Просмотр логов транзакций и списания расходников (кто сколько потратил, просмотр скриншотов).

Модуль 6: Публичный Лендинг и Расписание
Главная страница (/): Доступна без авторизации. Содержит название "Aqbobek Technopark" и кнопки входа.

Расписание: Публичная таблица таймслотов на текущий день (например, 14:00-15:30: "Кружок робототехники", 16:00-18:00: "Свободное время").

Бронирование: Кнопка для учителей "Запросить бронь", которая отправляет заявку администратору (запись в таблицу booking_requests).
